Sistema de Recomendação de Produtos

Projeto de TCC: Desenvolvimento de um sistema que sugere produtos com base no histórico de navegação do usuário.

1. Configurar Ambiente 2. Criar Banco de Dados 3. Desenvolver Frontend 4. Implementar Recomendação 5. Conectar Backend e Frontend 6. Testar e Ajustar

1. Configurar Ambiente

Nesta etapa, configuramos o ambiente de desenvolvimento. Optamos por utilizar o Spring Boot com Java para o backend devido à sua robustez e ampla adoção no mercado. As etapas incluem:

  • Instalar o JDK (Java Development Kit).
  • Configurar uma IDE, como IntelliJ IDEA ou Eclipse.
  • Criar um novo projeto Spring Boot utilizando o Spring Initializr, adicionando dependências como Spring Web, Spring Data JPA e MySQL Driver.

2. Criar Banco de Dados

Para armazenar informações de produtos, usuários e histórico de compras, utilizamos o MySQL. As atividades desta etapa incluem:

  • Instalar e configurar o MySQL Server.
  • Criar um banco de dados denominado "recomendacao".
  • Definir as tabelas necessárias, como "produtos", "usuarios" e "historico_compras".

3. Desenvolver Frontend

O frontend é responsável pela interface com o usuário. Utilizamos HTML, CSS e JavaScript para criar uma interface amigável e responsiva. As tarefas incluem:

  • Desenvolver páginas HTML estruturadas semanticamente.
  • Aplicar estilos CSS para garantir uma identidade visual consistente.
  • Implementar interações com JavaScript para melhorar a experiência do usuário.

4. Implementar Recomendação

Para sugerir produtos relevantes aos usuários, implementamos um sistema de recomendação utilizando técnicas de inteligência artificial. As ações desta fase são:

  • Analisar o histórico de navegação e compras dos usuários.
  • Desenvolver algoritmos de recomendação utilizando bibliotecas como TensorFlow.js.
  • Testar e validar os modelos de recomendação para garantir precisão nas sugestões.

5. Conectar Backend e Frontend

Para que o frontend e o backend se comuniquem de forma eficiente, implementamos uma API RESTful. As etapas envolvem:

  • Desenvolver endpoints no backend para fornecer dados de produtos e receber informações do usuário.
  • Consumir esses endpoints no frontend utilizando fetch API ou bibliotecas como Axios.
  • Garantir a segurança e integridade dos dados durante a comunicação.

6. Testar e Ajustar

Após o desenvolvimento, realizamos testes para assegurar a qualidade do sistema. As atividades incluem:

  • Executar testes unitários e de integração no backend.
  • Realizar testes de usabilidade e responsividade no frontend.
  • Coletar feedback de usuários e ajustar o sistema conforme necessário.
